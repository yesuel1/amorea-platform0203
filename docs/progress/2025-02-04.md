#### 4. 개발 환경 설정 (`next.config.ts`)
- Firebase Studio cross-origin 허용 설정

#### 5. 카운셀러 상세 프로필 페이지 (`/counselors/[id]`)
- 공유 데이터 분리 (`src/data/counselors.ts`) — Counselor 타입 인터페이스
- 동적 라우트로 4명의 개별 상세 페이지 자동 생성 (generateStaticParams)
- 각 페이지 구성:

| 섹션 | 내용 |
|------|------|
| 히어로 | 다크 퍼플 배경, 이모지 프로필, 이름/역할 |
| 소개 | 상세 경력 + 철학 인용문 |
| 사이드바 | 전문분야 태그 + 자격/인증 목록 |
| 프로그램 | 카운셀러별 맞춤 프로그램 3개 |
| 후기 | 별점 + 고객 후기 2개 |
| CTA | 상담 예약 + 다른 카운셀러 보기 |

- 라우트:
  - `/counselors/kim-minjeong` — 김민정 (저속노화 스킨케어)
  - `/counselors/lee-sujin` — 이수진 (역노화 메이크업)
  - `/counselors/park-younghee` — 박영희 (이너뷰티)
  - `/counselors/choi-eunyoung` — 최은영 (바디&웰니스)

#### 6. 카운셀러 지원 폼 페이지 (`/apply`)
- Client Component, 3단계 스텝 폼 위저드:

| 단계 | 항목 |
|------|------|
| Step 1: 기본 정보 | 이름*, 연락처*, 이메일, 연령대*(버튼), 직업*(8개 선택) |
| Step 2: 관심 & 동기 | 관심 분야*(복수선택 6개), 뷰티 경험, 지원 동기*(텍스트) |
| Step 3: 추가 정보 | 선호 시간대, SNS 계정, 지원 요약, 개인정보 동의* |

- 프로그레스 바, 필수 항목 미입력 시 버튼 비활성화
- 제출 완료 축하 화면
- 메인 페이지 CTA 버튼 3곳에서 `/apply`로 연결

#### 7. 프로젝트 문서화
- `CLAUDE.md` — Claude Code 프로젝트 가이드 (빌드 명령어, 아키텍처, 디자인 시스템)
- `docs/progress/` — 날짜별 진행현황 기록

### 핵심 메시지 구조
```
건강하게 아름다워지면서 (저속노화/역노화)
  → 경험을 공유하고 (퍼스널 브랜딩)
    → 추가 수익 + 자아실현 (리쿠르팅)
```

### 타겟 오디언스
- 육아맘: 나만의 시간·수익·아름다움
- 자영업자: 기존 사업 + 뷰티 시너지
- N잡러: SNS 콘텐츠 + 퍼스널 브랜딩
- 뷰티 러버: 좋아하는 것을 직업으로
